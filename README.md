# Manual QA Testing Project – SauceDemo

## Project Overview
This repository contains a manual testing project conducted on the SauceDemo web application (https://www.saucedemo.com).

The purpose of this project is to demonstrate practical manual QA skills including:
- Exploratory testing
- Test scenario creation
- Test case design
- Bug reporting
- Severity & priority assessment
- Evidence documentation

---

## Application Under Test
SauceDemo (E-commerce demo application)

Core areas tested:
- Login functionality
- Product inventory
- Cart behavior
- Checkout process
- Input validation

---

## Test Documentation

Located in `/docs`:

- `exploratory-session-01.md` – Initial exploratory findings
- `test-scenarios.md` – High-level test scenarios
- `test-cases.md` – Detailed manual test cases
- `test-summary-report.md` – Execution summary

## Bug Tracking

Defects are tracked using GitHub Issues with:
- Structured bug report template
- Severity classification
- Priority classification
- Status tracking
- Screenshot evidence

Example bugs:
- Checkout accepts non-numeric ZIP code
- Login input handling with trailing spaces
